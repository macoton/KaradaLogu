# KaradaLogu

シンプルなHTML/JavaScriptアプリケーション。

## 機能

- 入力画面: 
  - テキストボックスに文字列を入力し、**Enterキー**で記録
  - 全角数字（例：１２３）は自動的に半角（123）に変換されます
  - 3桁の数字を入力すると、入力値÷10を体温として記録
  - 9桁の数字を入力すると、前3桁を最高血圧、次3桁を最低血圧、最後3桁を脈拍として記録
  - 上記以外の数字列（例: 12、1234、0000000001など）も含め、数字と文字の混合や純粋な文字列はすべて「メモ」として扱われます。
  - メモは日時とともに保存され、入力画面にはボタン、出力画面にはテーブル形式で一覧表示されます。ボタンを押すだけで同じメモが再度記録されます。
  - メモの文字数制限はありません。

- 出力画面: 記録した体温・血圧・その他メモの一覧を表示（日時はローカル表記で“T”や“Z”は含みません）
- 設定画面（「記録を確認」）: 
  - データの全削除（操作前に確認ダイアログが出ます）
  - 記録をJSON形式でエクスポート
  - JSONファイルからインポート（読み込み時に「破棄して上書き」「マージ」「キャンセル」を選択可能）
  - Google Drive機能
    - Google Identity Services（GIS）を利用したOAuth2トークン取得
      - 認証ボタン押下でGISの `initTokenClient` を初期化し、アクセストークンを取得
      - APIキー/クライアントIDは `/cgi-bin/apikey.cgi` と `/cgi-bin/clientid.cgi` から読み込み
      - トークンは `accessToken` 変数に保持され、認証完了時にコンソールに出力される
    - 認証後「Driveにエクスポート」「Driveから読み込み」ボタンが表示される
    - エクスポートでは既存ファイルを更新、なければ新規作成（`drive.file` と `appDataFolder` の両スペースを検索）
    - 読み込みも同様に両スペースから探し、見つかったファイルをJSONとして読み込んで上書き/マージ/キャンセル選択
    - 読み込みに失敗した際はブラウザ開発者ツールのコンソールログを確認すると検索クエリやHTTPステータスが出力される


## 実行方法

1. リポジトリをクローン
2. `index.html` をブラウザで開く

データはブラウザの `localStorage` に保存されます。

（過去に不正なデータが残っていても自動的に無効化され、エラーになりません。）
